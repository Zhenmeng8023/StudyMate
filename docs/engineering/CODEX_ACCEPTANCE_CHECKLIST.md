# StudyMate 工作包验收与提交清单

## A. 开始前

- [ ] 已阅读 `AGENTS.md`、项目上下文、路线图和任务队列。
- [ ] 已执行 `git status`，确认不会覆盖用户未提交修改。
- [ ] 已定位相关模块的路由、handler、service、repository、migration、前端 API 调用和页面。
- [ ] 已明确本次仅处理一个工作包，并写出不做事项。
- [ ] 已识别数据迁移、权限、跨存储、异步任务与兼容性风险。

## B. 设计与实现

- [ ] API 契约（DTO、状态码、分页/筛选、错误）明确且前后端一致。
- [ ] 访问控制与资源归属校验完整；管理员能力有角色校验。
- [ ] 不把密钥、账号、生产 DSN、API Key 提交进仓库。
- [ ] 数据库变更有 migration、回滚/兼容考虑和索引评估。
- [ ] 多存储/异步任务有幂等、失败状态、重试或补偿策略。
- [ ] 图谱逻辑优先沉入 core/纯函数，而不是继续堆进页面组件。
- [ ] 新增复杂前端逻辑已拆分为合适的 page/component/hook/api/store/type。
- [ ] 变更没有扩大 `App.tsx`/`App.vue` 的聚合职责。

## C. 测试与验证

- [ ] 后端运行 `gofmt`。
- [ ] 运行相关 Go 单元测试；关键 handler/service 至少有新增或更新测试。
- [ ] 用户端运行 lint/typecheck/build（以仓库脚本为准）。
- [ ] 管理端运行 lint/typecheck/build（以仓库脚本为准）。
- [ ] 若涉及图谱，验证创建、编辑、保存、刷新/恢复、撤销/重做、导入/导出中的相关路径。
- [ ] 若涉及搜索，验证权限隔离、分页、筛选、空态、错误态和跨类型跳转。
- [ ] 若涉及 AI/任务，验证失败状态、重试路径、用量/审计记录。
- [ ] 若涉及迁移，至少在干净数据库或可重复环境验证 migrate 路径。

## D. 文档与交付

- [ ] 更新 `CODEX_BACKLOG.md` 状态与执行记录。
- [ ] 必要时更新 README、PROJECT_LOG、接口或架构文档。
- [ ] 提交粒度单一、信息清楚，不混入无关格式化。
- [ ] PR/最终汇报已说明：改动、测试、未执行项、风险、迁移、下一步。

## E. 特别禁止项

- [ ] 没有以“临时 mock/placeholder”冒充已完成的后台治理能力。
- [ ] 没有为了新功能而绕过权限、数据归属或错误处理。
- [ ] 没有未经兼容策略直接更改已被前端使用的 API。
- [ ] 没有一次性重写图谱画布或大范围迁移所有前端结构。
- [ ] 没有跳过 P0/P1 而开始课程、协作、桌面端或工程图谱大项目。
