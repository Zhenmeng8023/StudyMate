# v0.5.0 图谱画布 MVP

## 版本定位

完成学伴图谱的核心差异化能力：单人知识图谱画布。此版本是产品从学习资料平台转向知识组织平台的关键节点。

## 设计约束

- 使用 Canvas 2D，不引入 WebGL。
- 使用自研轻量图对象模型，不依赖展示型 mindmap 库。
- 第一版只做单人图谱，不做多人协作和 CRDT。
- 图谱内容写入 MongoDB，业务索引、权限和版本写入 MySQL。

## 交付切片

| 切片 | 交付内容 |
| --- | --- |
| 数据 | `graphs`、`graph_permissions`、`graph_versions`、`graph_documents`、`graph_snapshots` 预留 |
| 后端 | 图谱 CRUD、图谱文档读取保存、批量保存、基础权限校验 |
| 前台 | 图谱列表页、图谱画布页、节点工具栏、基础属性面板、自动保存状态 |
| 图谱核心 | Camera、节点、边、视口、选择态、拖拽、框选、连线、历史操作模型 |
| 文档 | 图谱文档结构、图谱 API 摘要、画布交互验收清单 |

## API 范围

- `GET /api/v1/graphs`
- `POST /api/v1/graphs`
- `GET /api/v1/graphs/:id`
- `PUT /api/v1/graphs/:id`
- `DELETE /api/v1/graphs/:id`
- `POST /api/v1/graphs/:id/batch-save`

## 验收标准

- 用户可以创建一张图谱。
- 用户可以添加 20 个以上节点并流畅拖拽。
- 用户可以选择、框选、连接和删除节点。
- 用户可以创建直线边和曲线边。
- 用户可以保存图谱，刷新页面后内容能恢复。
- 撤销和重做覆盖新增、移动、删除、连线。

## 交给下一版本

- 稳定的图谱文档模型。
- 可扩展的节点、边和视口状态。
- 基础历史记录和自动保存能力。
- 可继续增加分组、迷你地图、搜索定位和导入导出的画布骨架。

## 暂不做

- 多人协作。
- CRDT。
- 高级工程图规范模式。
- 复杂自动布局。
- 插件系统。
- Tauri 桌面端。
