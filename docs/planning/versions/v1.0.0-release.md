# v1.0.0 正式发布版

## 版本定位

达到可公开演示和小规模试用的产品质量。重点从“功能堆叠”转向稳定性、体验一致性、文档完整性和闭环可用性。

## 发布范围

| 范围 | 发布要求 |
| --- | --- |
| 学习闭环 | 资料阅读、笔记沉淀、图谱组织、AI 辅助、卡片复习、社区分享可串起来 |
| 后台治理 | 用户管理、内容审核、资料管理、图谱模板、AI 任务、系统配置可用 |
| 数据与部署 | MySQL、MongoDB、Redis、Meilisearch 部署说明和迁移脚本完整 |
| 质量 | 核心路径回归测试、基础性能检查、安全检查和文件访问检查完成 |
| 文档 | 本地开发、部署、数据库迁移、API 概览、演示脚本和演示数据完整 |

## 交付切片

| 切片 | 交付内容 |
| --- | --- |
| 后端 | 核心模块接口稳定、错误码规范、统一响应结构、数据迁移脚本完整、基础安全策略 |
| 前台 | 工作台、社区、资料库、PDF 阅读器、笔记、图谱、卡片复习、AI 助手体验统一 |
| 后台 | 用户管理、内容审核、资料管理、图谱模板、AI 任务、系统配置 |
| 数据 | 演示数据、备份恢复建议、生产环境 `.env` 模板 |
| 文档 | 本地开发、部署、迁移、API 概览、演示脚本、排错说明 |

## 验收标准

- 新用户可以从注册开始完成完整学习闭环。
- 管理员可以治理公开内容和查看 AI 使用情况。
- 核心路径无阻塞级错误。
- 构建、测试、启动文档可复现。
- 至少准备一套完整演示数据。
- 核心页面具备空状态、加载态、错误态。
- 移动端具备基础适配，不出现阻塞级布局问题。

## 发布前检查

- `go test ./...` 通过。
- 前台和后台构建通过。
- 数据库迁移可从空库执行。
- 核心接口鉴权、权限和文件访问校验通过。
- AI 用量、失败和超限状态可见。
- 演示脚本可以按步骤复现。

## 暂不做

- 大规模商业化部署。
- 多人实时协作。
- 桌面端 Tauri。
- 移动原生 App。
- 插件市场。
- 完整 RAG 向量知识库。

# v1.0.0 Release And Rollback Checklist

## Release status

- Date: 2026-06-01
- Scope: Web user site, Web admin, backend API.
- Non-blockers: Courses/LMS, Tauri desktop, realtime collaboration, PWA offline, vector search, full English localization.
- Source locale: `zh-CN`; `en-US` remains placeholder-only for v1.
- Search: MySQL fallback is the v1 default; vector search is not required for release.
- Review scheduler: SM-2 is the v1 default behind the backend `Scheduler` interface.

## Final verification gate

Run these before tagging:

```powershell
npm run ci
npm run test:coverage
npm run verify:secrets
npm run verify:backend:format
npm run verify:config-safety
git diff --check
git status --short
```

Secret scan:

```powershell
npm run verify:secrets
```

Expected notes:
- Example env names and docs may match the scan, but no concrete production secret should be committed.
- `npm run test:coverage` records summary evidence; changed packages should target 80% focused coverage or record gaps in `PROJECT_LOG.md`.

| Variable | Required | Example | Notes |
| --- | --- | --- | --- |

| `DB_DRIVER` | Yes | `mysql` | Only MySQL is supported for v1. |
| `MYSQL_DSN` | Yes | `user:pass@tcp(host:3306)/studymate?charset=utf8mb4&parseTime=True&loc=Local` | Keep out of git. |
| `MONGO_URI` | Yes | `mongodb://host:27017` | Content read models and graph documents. |
| `MONGO_DATABASE` | Yes | `studymate_content` | Must match migration target. |

| `ADMIN_BOOTSTRAP_USERNAME` | Yes for first deploy | `admin` | Used only to ensure bootstrap admin. |
| `ADMIN_BOOTSTRAP_EMAIL` | Yes for first deploy | `admin@studymate.local` | Replace in production. |
| `ADMIN_BOOTSTRAP_PASSWORD` | Yes for first deploy | `<secret-manager-value>` | Do not commit. |
| `NOTE_READ_MODEL` | Yes | `mysql_primary` | `mongo_primary` is supported with MySQL fallback. |

Additional release notes:

- Backend startup no longer accepts implicit fallback values for `MYSQL_DSN` or `JWT_SECRET`; release environments must set them explicitly.
- Default Playwright preview ports are `44173` and `44174`, and can be overridden with `PLAYWRIGHT_USER_PORT` / `PLAYWRIGHT_ADMIN_PORT` for unusual CI runners.

## Migration order

1. Back up MySQL, MongoDB, and upload storage.
2. Run MySQL migrations with `cd backend; go run ./cmd/migrate`.
3. Run Mongo collection/index initialization for a new database with `backend/internal/migrations/mongo/001_init_content_collections.js`.
4. Backfill historical note documents with `cd backend; go run ./cmd/backfill-note-documents -limit 500` when migrating an existing MySQL-only note dataset.
5. Start backend, user frontend, and admin frontend.
6. Confirm `/health` and `/api/v1/health`.

## Demo data steps

1. Bootstrap an admin and create one normal user.
2. Upload one PDF material and approve it in admin.
3. Open the reader, create one annotation with `rects`, and generate a note.
4. Create or update a graph from note/annotation context.
5. Generate or manually create cards, review one due card, and confirm SM-2 writeback.
6. Search for the material/note/graph/card from `/search?q=...`.
7. Create a share link for one material/note/graph/deck and open `/share/:token`.
8. Open admin modules: users, reports, tags, AI tasks/usage, audit logs, files.

## Rollback steps

1. Stop frontend and backend deploys.
2. Restore the previous application artifact or container image.
3. If `004_share_links.sql` was applied and must be reverted before any share links matter, run `004_share_links.down.sql`; otherwise keep the table because it is additive and isolated.
4. If `003_align_current_tables.sql` added `pdf_annotations.rects`, old clients remain compatible and the column may stay in place.
5. Restore MySQL/Mongo/upload backups if data corruption, not just application rollback, is confirmed.
6. Set `NOTE_READ_MODEL=mysql_primary` during rollback to keep note reads anchored on MySQL.
7. Re-run `npm run ci` against the restored revision before re-opening traffic.

## Known v1 non-blockers

- Full English localization is not complete; `en-US` is a placeholder dictionary framework.
- Vector search and Meilisearch adapter are deferred; MySQL fallback is the release path.
- Courses/LMS, Tauri desktop, realtime collaboration, and PWA offline are deferred.
- Coverage summaries are recorded locally; enforcement remains milestone-focused until CI coverage thresholds are hardened.
