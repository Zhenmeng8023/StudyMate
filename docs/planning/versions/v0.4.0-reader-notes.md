# v0.4.0 阅读笔记版

## 版本定位

完成资料阅读到笔记沉淀的核心路径，为后续图谱节点、卡片生成和 AI 引用提供可信来源。当前阶段应优先收束本版本，不要过早做深 AI、卡片和工程图。

## 交付切片

| 切片 | 交付内容 |
| --- | --- |
| 数据 | `notes`、`note_folders`、`note_tags`、`note_relations`、`reading_progress`、`note_documents`、`note_snapshots`、`pdf_annotations` |
| 后端 | PDF 阅读状态、批注保存、笔记索引、笔记内容保存、版本历史、版本恢复、来源关系 |
| 前台 | PDF 阅读页、高亮和批注面板、书签、笔记列表、富文本编辑页、版本历史面板 |
| 后台 | 资料审核与内容治理沿用 `v0.3.0`，不新增复杂后台 |
| 文档 | 笔记数据流、MongoDB 内容文档说明、阅读和笔记接口摘要 |

## 验收标准

- 用户可以从资料详情进入 PDF 阅读页。
- 用户可以记录阅读进度、添加书签和批注。
- 用户可以从资料或批注创建关联笔记。
- 用户可以编辑并保存富文本笔记。
- 每次保存可产生可恢复的历史版本。
- 笔记内容写入 MySQL 索引和 MongoDB 内容文档/快照。

## 交给下一版本

- 可信的资料、批注、笔记来源关系。
- 可转化为图谱节点草稿的数据结构。

- PDF 批注和摘录池的基础数据。
