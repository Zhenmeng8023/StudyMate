# v0.6.0 图谱产品化版

## 版本定位

`v0.6.0` 的定位已经从“规划阶段”推进为“强 MVP 后的产品化收口”。当前图谱工作区已经具备搜索定位、来源泳道、来源摘要、AI 落点预览、Markdown/Mermaid 导入、StudyMate JSON 导入导出、PNG/SVG 导出和快照恢复基础；下一步重点是稳定、治理和可达性，不是推翻重做。

## 已有基础

- Canvas 图谱工作区、节点、边、分组、视口和选择态。
- 图谱保存、快照恢复、导入和导出入口。
- `.smtg` / `application/vnd.studymate.graph+json` 文件格式，包含 schemaVersion、nodes、edges、groups、viewport、theme 和 metadata。
- 搜索定位与来源泳道布局。
- 来源摘要可展示资料、批注、笔记、卡片和 AI 草稿等来源对象。
- AI 预计落点已经收敛为一次性预览状态，不再污染图谱文档。
- 主导航从图谱离页恢复为流畅 SPA 切换。
- `@studymate/graph-core` 已沉淀文档规范化、验证规则、学习模板、history label、JSON 导入导出和 200/300/20 基准夹具。
- 扩展节点已具备 URL、图片、公式和 PDF 锚点的结构化 metadata 编辑入口，保持与通用标题、描述、来源、颜色、强调和尺寸编辑并存。
- `.smtg` JSON 导入在用户端会阻断 schemaVersion、重复 ID、非法尺寸、悬挂边和无效来源 target 等 error 级问题；后端保存和导入同样拒绝 error 级结构问题。
- 200 节点、300 边、20 分组的 E2E smoke 已覆盖打开和手动保存状态。
- 图谱工作区继续从大型 controller 中拆出 header、来源 rail、toolbar、stage status、stage world、minimap、空态、来源摘要、快照恢复和卡片草稿纯视图组件，并补组件级可达性/交互测试；节点/连线详情编辑和画布交互状态机仍是后续拆分重点。
- `@studymate/graph-core` 已完成第一轮模块化拆分，`index.ts` 保留为 barrel，核心能力分布在 model、source、mutations、validation、file-format、history、templates、fixtures、selection 和 viewport 等小模块中。
- 用户端图谱页面级回归已覆盖节点标题、URL metadata、边标签、直线/曲线切换和保存 payload；E2E 200 节点 smoke 已扩展 JSON 导入失败、快照恢复失败、快捷键面板和来源反链跳转。
- 后端 graph handler 已把无效图谱请求统一修正为 400，服务端验证规则已兼容多目标边 `metadata.targetNodeIds` 的 JSON 与内部构造形态。
- 右侧 rail 的节点/连线详情编辑已拆出为 `GraphWorkspaceSelectionPanel`，用组件测试覆盖单节点元数据、边样式、多选整理、分组编辑和空态提示；大型 controller 后续剩余重点是画布交互状态机和导入导出/数据加载 hook。
- 右侧 rail 的导入与校验 UI 已拆出为 `GraphWorkspaceImportPanel`，用组件测试覆盖导入模式、导入内容、导入/校验按钮和验证问题展示；导入执行分支后续继续下沉到独立 hook。
- 图谱保存、自动保存、离页保护、快照列表和快照恢复已拆出为 `useGraphWorkspacePersistence`，用 hook 测试锁定保存成功刷新快照、恢复快照重置 history、快照 API 失败时保留编辑状态；大型 controller 继续下降到约 1938 行。
- Markdown/Mermaid 远端导入、StudyMate JSON 本地导入和 PNG/SVG/JSON 导出已拆出为 `useGraphImportExport`，用 hook 测试锁定 JSON 阻断错误、远端导入刷新快照、安全文件名和导出失败状态；大型 controller 继续下降到约 1857 行。

## 1.0 前必须收口

| 能力 | 交付要求 |
| --- | --- |
| 图谱模板 | 提供学习资料梳理、读书笔记、概念网络、复习卡片准备等基础模板 |
| 验证规则面板 | 展示孤立节点、缺来源节点、重复标题、悬挂边、跨折叠分组边、空分组、非法尺寸和无效来源 target 等问题 |
| autosave/dirty | 明确 pending、dirty、saved、failed 状态；离页前不丢失用户编辑 |
| Undo/Redo 边界 | 持久化边界清晰，保存、导入、恢复、模板应用形成可解释历史点 |
| 来源反链 | 从图谱节点能回到资料、批注、笔记、PDF 页或 AI 草稿 |
| 键盘可达菜单 | 常用图谱操作可通过键盘聚焦和触发 |
| 无障碍标签 | 画布、工具栏、节点列表、设置面板具备可读标签 |
| 设置面板 | 把导入导出、显示偏好、自动保存和性能提示收敛到清晰区域 |

## 性能回归场景

基准数据：

- 200 个节点
- 300 条边
- 20 个分组

必须验证：

- 首次打开可用。
- 拖动不持续卡死。
- 保存能给出明确成功、失败或 pending 状态。
- PNG/SVG/JSON 导出可完成。

建议预算见 [VERSION_PLAN.md](../VERSION_PLAN.md#4-建议性能预算)。

## 验收标准

- 用户能长期整理图谱，而不是只创建一次性画布。
- 用户能理解图谱内容来自哪些资料、批注、笔记、卡片或 AI 草稿。
- 图谱离页、返回、保存失败、导入失败和导出失败都有明确状态。
- 图谱核心回归测试、前端构建和后端测试通过。

## 暂不做

- 多人实时协作。
- CRDT。
- WebGL 重写。
- PlantUML / Structurizr 深度集成。
- 插件市场。
- 复杂自动布局引擎。
- Project Graph `.prg` 兼容；后续如需要，以转换器形式进入单独版本。
