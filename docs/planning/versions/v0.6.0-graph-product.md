# v0.6.0 图谱产品化版

## 版本定位

`v0.6.0` 的定位已经从“规划阶段”推进为“强 MVP 后的产品化收口”。当前图谱工作区已经具备搜索定位、来源泳道、来源摘要、AI 落点预览、Markdown/Mermaid 导入、StudyMate JSON 导入导出、PNG/SVG 导出和快照恢复基础；下一步重点是稳定、治理和可达性，不是推翻重做。

## 已有基础

- Canvas 图谱工作区、节点、边、分组、视口和选择态。
- 图谱保存、快照恢复、导入和导出入口。
- `.smtg` / `application/vnd.studymate.graph+json` 文件格式，包含 schemaVersion、nodes、edges、groups、viewport、theme 和 metadata。
- 搜索定位与来源泳道布局。
- 来源摘要可展示资料、批注、笔记、卡片和 AI 草稿等来源对象。
- AI 预计落点已经收敛为一次性预览状态，不再污染图谱文档。
- 主导航从图谱离页恢复为流畅 SPA 切换。
- `@studymate/graph-core` 已沉淀文档规范化、验证规则、学习模板、history label、JSON 导入导出和 200/300/20 基准夹具。

## 1.0 前必须收口

| 能力 | 交付要求 |
| --- | --- |
| 图谱模板 | 提供学习资料梳理、读书笔记、概念网络、复习卡片准备等基础模板 |
| 验证规则面板 | 展示孤立节点、缺来源节点、重复标题、悬挂边、跨折叠分组边、空分组、非法尺寸和无效来源 target 等问题 |
| autosave/dirty | 明确 pending、dirty、saved、failed 状态；离页前不丢失用户编辑 |
| Undo/Redo 边界 | 持久化边界清晰，保存、导入、恢复、模板应用形成可解释历史点 |
| 来源反链 | 从图谱节点能回到资料、批注、笔记、PDF 页或 AI 草稿 |
| 键盘可达菜单 | 常用图谱操作可通过键盘聚焦和触发 |
| 无障碍标签 | 画布、工具栏、节点列表、设置面板具备可读标签 |
| 设置面板 | 把导入导出、显示偏好、自动保存和性能提示收敛到清晰区域 |

## 性能回归场景

基准数据：

- 200 个节点
- 300 条边
- 20 个分组

必须验证：

- 首次打开可用。
- 拖动不持续卡死。
- 保存能给出明确成功、失败或 pending 状态。
- PNG/SVG/JSON 导出可完成。

建议预算见 [VERSION_PLAN.md](../VERSION_PLAN.md#4-建议性能预算)。

## 验收标准

- 用户能长期整理图谱，而不是只创建一次性画布。
- 用户能理解图谱内容来自哪些资料、批注、笔记、卡片或 AI 草稿。
- 图谱离页、返回、保存失败、导入失败和导出失败都有明确状态。
- 图谱核心回归测试、前端构建和后端测试通过。

## 暂不做

- 多人实时协作。
- CRDT。
- WebGL 重写。
- PlantUML / Structurizr 深度集成。
- 插件市场。
- 复杂自动布局引擎。
- Project Graph `.prg` 兼容；后续如需要，以转换器形式进入单独版本。
