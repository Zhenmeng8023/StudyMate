# UI-04 产品级界面重构

## 目标

本轮将 StudyMate 从“功能已接入、页面仍像开发面板”的状态，提升为以学习任务和治理任务为中心的产品界面。范围覆盖用户端全部路由和管理端全部模块；不改变既有 API、数据模型、图谱版本契约或权限语义。

## 截图审计结论

### 用户端图谱

- 资源栏、工具栏、画布和检查器同时常驻，画布成为四个区域中最弱的一块；右侧检查器在未选择节点时仍承载大量编辑字段。
- 命令栏、工具栏、状态条和画布卡片层层嵌套，视觉边框过多，优先级不清。
- 默认打开左右辅助区，使首次进入图谱不能先获得“浏览整个知识结构”的体验。

### 管理端

- 用户治理将一条记录排成大面积纵向卡片，数据密度低，无法扫描多个账号。
- 缺少搜索、表格列、状态徽章、记录详情和明确的治理上下文。
- 侧栏、顶部和内容区缺少统一的信息层级，内容区有大量无意义空白。

## 交互原则

1. **主任务优先**：阅读器突出正文，图谱突出画布，复习突出单张卡片，运营端突出队列和表格。
2. **按需出现的上下文**：Inspector 和 Drawer 仅在用户需要查看来源、属性或历史时展开；默认不侵占主区域。
3. **用密度区分任务**：学习端使用内容卡和可读正文，管理端使用可扫描表格和记录详情。
4. **统一状态语言**：加载、空数据、保存中、冲突和错误有一致的色彩、边距和行动入口。
5. **响应式而非硬缩放**：桌面端可并列，笔记本端改为覆盖式 Dock，移动端改为底部导航或单列。

## 页面模式

| 模式 | 路由 | 重构方向 |
|---|---|---|
| Standard | 工作台、资料库、社区、搜索、AI、设置 | 轻量侧栏、顶部命令栏、页面自有内容列 |
| Studio | 阅读器、笔记 | 左侧资源 / 主编辑区 / 右侧 Inspector |
| Canvas | 图谱 | 画布优先；资源和 Inspector 默认关闭 |
| Focus | 复习 | 单任务卡片，管理面板按需打开 |
| Admin | 全部管理路由 | 深色治理导航、运营概览、数据表格、记录 Inspector |

## 本轮落地

- 移除用户端 Standard 页面固定的全局 ContextPanel，避免所有页面强制出现与当前任务无关的说明。
- 用户端侧栏改为分组导航，顶部改为路径、页面名、全局搜索和账户操作。
- 图谱默认全画布进入；选择节点或检测到冲突时，自动打开相应 Inspector；修复左右面板并列时的 Grid 列定位。
- 阅读、笔记、复习保留 FE-03 的工作区结构，并用同一套视觉 token 收口。
- 资料库、社区、搜索、AI、设置、登录和工作台统一卡片密度、边距、状态和按钮层级。
- 管理端从“卡片列表”升级为治理工作台：概览指标、审核数据表、记录搜索、字段表格和详情 Inspector。

## 验收尺寸

| 页面 | 桌面 | 笔记本 | 手机 |
|---|---|---|---|
| 图谱 | 画布为主，Drawer/Inspector 可并列 | Drawer 覆盖画布 | 全屏 Drawer + 底部导航 |
| 阅读/笔记 | 三段工作区 | 两侧 Dock 覆盖 | 单列、按需打开面板 |
| 资料/社区 | 2–3 列信息架构 | 自动降为 2 列或 Inspector 下移 | 单列 |
| 管理端 | 侧栏 + 表格 + Inspector | 窄导航 + 表格横向滚动 | 底部导航 + 单列卡片/表格滚动 |

## 保持不变的边界

- `GraphDocument`、图谱节点/边/分组结构、图谱版本推进和 `409 graph_version_conflict` 不改动。
- 阅读、笔记、卡片、AI、后台 API URL 与请求格式不改动。
- 所有重构为展示、布局和前端交互层改造；业务数据仍以现有后端为准。
